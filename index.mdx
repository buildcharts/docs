---
title: "BuildCharts"
description: "Generate Docker Buildx Bake plans from declarative build metadata and OCI-hosted charts."
---

`buildcharts` is a .NET tool for defining build intent in `build.yml` and turning that metadata into a Docker Buildx Bake plan. You keep build definitions in your repo, pull reusable chart templates from an OCI registry, and run the same generated plan locally or in CI.

## What you define

```text
build.yml                     # Targets, variables, plugins, and type matrices
charts/buildcharts/Chart.yaml # Chart aliases and OCI dependencies
charts/buildcharts/Chart.lock # Optional digests for reproducible builds
```

## What BuildCharts generates

- `.buildcharts/docker-bake.hcl` from your metadata and chart dependencies
- `.buildcharts/output/<type>` when non-docker targets export artifacts through Bake
- Optional summaries and plugin output under `.buildcharts`

## Workflow

1. Create or scaffold `build.yml` and `charts/buildcharts/Chart.yaml`.
2. Run `buildcharts update` if you want a fresh `Chart.lock`.
3. Run `buildcharts generate`.
4. Run `docker buildx bake --file .buildcharts/docker-bake.hcl`.

## Why teams use it

- Keep CI logic in versioned OCI charts instead of copying pipeline YAML across repos.
- Run the same generated build plan locally and in CI.
- Describe intent with aliases such as `build`, `test`, `nuget`, and `docker`.
- Extend generation with built-in plugins such as `NuGetAuthenticate@v1` and `TestcontainersDinD@v1`.
