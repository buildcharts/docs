## Prerequisites

- Docker with `buildx`
- .NET 10 SDK (only if you install BuildCharts as a .NET tool)

## Install as a global tool

```bash
dotnet tool install --global dotnet-buildcharts
```

This makes the `buildcharts` command available in your shell.

## Install in a local tool manifest

Use this when you want the repository to pin the CLI version:

```bash
dotnet new tool-manifest
dotnet tool install dotnet-buildcharts
```

Then restore the tool inside the repo with:

```bash
dotnet tool restore
```

## Avoid external build-time dependencies (optional)

When you run `buildcharts update` or `buildcharts generate`, BuildCharts resolves chart dependencies from an OCI container registry (for example Docker Hub).

If you want to avoid relying on an external registry at build time, consider forking the chart ecosystem and publishing the charts to your own registry or organization. Then update `charts/buildcharts/Chart.yaml` to point to your internal `repository: oci://...`.

See [Charts](/charts) for details.

## Verify the installation

```bash
buildcharts version
```
