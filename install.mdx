## Prerequisites

- Docker with `buildx`
- .NET 10 SDK (only if you install BuildCharts as a .NET tool)

## Install as a global tool

```bash
dotnet tool install --global dotnet-buildcharts
```

This makes the `buildcharts` command available in your shell.

## Install in a local tool manifest

Use this when you want the repository to pin the CLI version:

```bash
dotnet new tool-manifest
dotnet tool install dotnet-buildcharts
```

Then restore the tool inside the repo with:

```bash
dotnet tool restore
```

## Avoid external tool restore dependencies (optional)

If you install BuildCharts as a .NET tool, `dotnet tool install` and `dotnet tool restore` download the tool package from a NuGet feed (by default `nuget.org`).

If you want to avoid relying on external package feeds at build time, mirror or publish the tool package to your own NuGet feed (for example Azure Artifacts) and configure your `NuGet.config` (or `--add-source`) to use it. Another option is to install the tool once in your build image so CI does not need to download it on every run.

## Verify the installation

```bash
buildcharts version
```
