Plugins are the main extension point in BuildCharts.

Plugins run during `buildcharts generate`.

## What plugins do

Plugins can:
- prepare external state before generation
- write secret or debug files under `.buildcharts`
- patch the generated `docker-bake.hcl`

## How to enable them

Declare plugins at the top level of `build.yml`:

```yaml
plugins:
  - NuGetAuthenticate@v1
  - TestcontainersDinD@v1
```

BuildCharts runs plugins in declaration order and ignores unknown names with a warning.

## Built-in plugins

- [NuGetAuthenticate](/reference/plugin-nuget-authenticate)
- [TestcontainersDinD](/reference/plugin-testcontainers-dind)

Use this page for the plugin model and `build.yml` wiring, then use the plugin pages for behavior and environment details.

## Example

```yaml
version: v1beta

plugins:
  - TestcontainersDinD@v1

variables:
  - VERSION
  - COMMIT

targets:
  src/dotnet-testcontainers.csproj:
    type: [build, test]
    with:
      base: mcr.microsoft.com/dotnet/sdk:10.0
```
