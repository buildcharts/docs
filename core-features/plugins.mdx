---
title: "Plugins"
description: "Built-in plugins that extend generation before and after HCL creation."
---

Plugins hook into the `buildcharts` generation pipeline and run during `buildcharts generate`. They are the main extension point in BuildCharts for custom logic before and after the build configuration is produced.

## What plugins do

Plugins can:
- prepare external state before generation
- write secret or debug files under `.buildcharts`
- patch the generated `docker-bake.hcl`
- generate additional output files
- perform validation or environment setup

## How to enable them

Declare plugins at the top level of `build.yml`:

```yaml
plugins:
  - NuGetAuthenticate@v1
  - TestcontainersDinD@v1
```

When you run `buildcharts generate`, BuildCharts loads built-in plugins by name, removes duplicates, and runs them in declaration order. Unknown plugin names are skipped with a warning.

## Built-in plugins

- [NuGetAuthenticate@v1](/reference/plugin-nuget-authenticate)
- [TestcontainersDinD@v1](/reference/plugin-testcontainers-dind)

BuildCharts currently loads built-in plugins only. If you need a new plugin, you must extend the BuildCharts tool and ship it as part of your BuildCharts distribution.

Use this page for the plugin model and `build.yml` wiring, then use the plugin pages for behavior and environment details.

## Example

```yaml
version: v1beta

plugins:
  - TestcontainersDinD@v1

variables:
  - VERSION
  - COMMIT

targets:
  src/dotnet-testcontainers.csproj:
    type: [build, test]
    with:
      base: mcr.microsoft.com/dotnet/sdk:10.0
```
